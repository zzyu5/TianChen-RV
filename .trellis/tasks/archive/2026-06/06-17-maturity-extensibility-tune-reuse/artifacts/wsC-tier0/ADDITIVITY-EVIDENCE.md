## Record matching is canonical-integer exact

The generic revalidation compares knob VALUES by string against the candidate's
`std::to_string(int)` spelling (where the typed code used `getAsInteger`). On every
CANONICAL record line -- all the offline driver ever emits, and all the tests use
-- these are identical (proven by the byte-diff + the present/absent/stale lit
cases). They diverge only on non-canonical integer spellings (`factor=02`, `+2`),
where the generic path falls CLOSED to the static fallback -- I7-aligned, and out
of scope for this refactor.
